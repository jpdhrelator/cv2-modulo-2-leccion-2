Ejercicios Prácticos · El Lenguaje HTML

Cinco retos prácticos para aplicar lo aprendido en la presentación M2_L2 El Lenguaje HTML: estructura completa de un documento, semántica HTML5, enlaces e imágenes con sus atributos, listas anidadas, tablas básicas y formularios sencillos.

📋 Antes de empezar

  • Cada ejercicio se entrega como un archivo .html.
  • Trabaja en VS Code con la extensión Live Server.
  • Solo HTML — sin CSS ni JS aún.
  • Indenta correctamente y cierra todas las etiquetas (buena práctica del slide 46).
  • Antes de entregar, valida tu HTML en validator.w3.org.

Documento HTML5 completo

🎯 Cabecera del documento 🏷️ DOCTYPE meta title lang

Contexto

En el slide 10 vimos que un documento HTML válido necesita <!DOCTYPE html>, un <html> con dos hijos (<head> y <body>), y dentro del head un <meta charset> y un <title> (este último es la única etiqueta obligatoria del head, slide 11).

Qué debes hacer

  1. Crea un archivo ejercicio-01.html.
  2. Construye un documento HTML5 que incluya:
    • La declaración <!DOCTYPE html> al inicio (slide 47).
    • Un elemento <html> con atributo lang para español.
    • Dentro del <head>:
      • Un <meta> que indique la codificación UTF-8.
      • Un <meta> con el name="viewport" para móviles.
      • Un <meta> con una descripción corta (name="description").
      • Un <title> con el texto "Mi portafolio web".
    • Dentro del <body>:
      • Un encabezado <h1> con tu nombre completo.
      • Un párrafo con una breve presentación (3-4 oraciones).
  3. Añade al menos un comentario HTML (<!-- ... -->) explicando alguna parte del código.

Criterios de éxito

  • El documento pasa la validación del W3C sin errores.
  • En la pestaña del navegador se ve el título correcto.
  • Caracteres especiales como "ñ" y tildes se muestran sin errores raros.

Enlaces e imágenes con todos sus atributos

🎯 Hipertexto y multimedia 🏷️ a img href src alt target download

Contexto

Los slides 21–27 cubrieron en detalle las dos etiquetas que dan nombre a HTML (HyperText) y la web visual: <a> con sus atributos href, target, rel, download; e <img> con src, alt, width, height.

Qué debes hacer

Crea un archivo ejercicio-02.html titulado "Recursos de aprendizaje para front-end". La página debe contener:

  1. Un encabezado principal con el título.
  2. Un párrafo introductorio.
  3. Una sección con al menos 4 enlaces:
    • Un enlace a MDN Web Docs que se abra en una pestaña nueva.
    • Un enlace a W3C que se abra en la misma pestaña (uso explícito de target="_self").
    • Un enlace a un PDF que se descargue automáticamente (atributo download).
    • Un enlace por correo electrónico que abra el cliente de correo (pista: investiga mailto:).
  4. Una galería con 3 imágenes:
    • Cada imagen con su src, su alt descriptivo y sus width y height.
    • Cada imagen envuelta en un enlace que apunte a la imagen en tamaño completo y se abra en pestaña nueva.
  5. Un comentario antes de la galería explicando qué hace ese bloque.

Criterios de éxito

  • Todos los enlaces funcionan según lo descrito.
  • Todas las imágenes cargan y tienen un alt útil (no "imagen 1").
  • El enlace de descarga efectivamente fuerza la descarga.

Listas anidadas: el menú de un restaurante

🎯 Jerarquía con listas 🏷️ ul ol li

Contexto

En los slides 29–33 vimos las listas no ordenadas (<ul>), las ordenadas (<ol>) y, lo más importante, cómo anidar listas para representar jerarquías reales.

Qué debes hacer

Crea un archivo ejercicio-03.html con el menú de un restaurante ficticio:

  1. Un encabezado principal con el nombre del restaurante.
  2. Una lista no ordenada de categorías del menú (al menos 4: Entradas, Platos principales, Postres, Bebidas).
  3. Dentro de cada categoría debe haber una lista anidada con al menos 3 platos.
  4. Para "Platos principales", cada plato debe contener a su vez una lista no ordenada con los ingredientes principales (al menos 3).
  5. Una sección final "Pasos para hacer la reserva" usando una lista ordenada con al menos 4 pasos (porque el orden importa).

Criterios de éxito

  • La anidación se hace correctamente: el sublista va dentro de un <li>, no entre <li>s.
  • Se usa <ul> para lo que no tiene orden y <ol> para lo que sí.
  • Al inspeccionar con DevTools, el árbol del DOM refleja la jerarquía visual.

Página completa con estructura semántica + tabla

🎯 Layout y datos tabulares 🏷️ header nav main aside footer table

Contexto

Los slides 14–19 mostraron el layout estándar con etiquetas semánticas y el slide 34 trató las tablas para mostrar datos tabulares (jamás para maquetar — "una aberración" según la presentación).

Qué debes hacer

Crea ejercicio-04.html con el horario de clases de un curso (real o ficticio). Prohibido usar <div>. La página debe contener:

  1. Un <header> con el nombre del curso y un subtítulo.
  2. Un <nav> con al menos 4 enlaces (pueden ir a #).
  3. Un <main> o <section> con:
    • Un párrafo introductorio explicando de qué semestre es el horario.
    • Una tabla con el horario semanal: filas = bloques horarios (al menos 5), columnas = días (lunes a viernes). Cada celda contiene la asignatura.
    • Primera fila con los nombres de los días y primera columna con los horarios.
  4. Un <aside> con un recuadro de "Recordatorios importantes".
  5. Un <footer> con el copyright y el año actual.

Criterios de éxito

  • No hay ni un solo <div> en el documento.
  • La tabla se construye con <table>, <tr> y <td> (pista: investiga también <thead> y <th> para los encabezados).
  • Cada bloque del layout está envuelto en su etiqueta semántica adecuada.
  • El documento valida sin errores en W3C.

Formulario de inscripción al curso

🎯 Inputs y controles 🏷️ form input select option button

Contexto

Los slides 35–44 cubrieron la etiqueta <form> con sus atributos action y method, y los 6 tipos de <input> vistos: text, password, checkbox, radio, submit y button, además de <select> con <option>.

Qué debes hacer

Crea ejercicio-05.html con un formulario de inscripción a un curso:

  1. Un encabezado principal con el título del formulario.
  2. Un párrafo introductorio.
  3. Un <form> que:
    • Tenga atributo action="procesar.php" y method="post".
    • Contenga los siguientes campos (cada uno con su atributo name):
      • Nombre completo (type="text").
      • Correo electrónico (type="text").
      • Contraseña (type="password").
      • Edad (puedes usar type="text").
      • Nivel de experiencia con un <select> de al menos 4 opciones.
      • Modalidad preferida: dos radio (Presencial / En línea) — deben compartir el mismo name.
      • Temas de interés: al menos 3 checkbox.
      • Un botón type="submit" con texto "Enviar inscripción".
      • Un botón type="button" con texto "Limpiar".
  4. Incluye comentarios HTML separando las secciones del formulario.

Criterios de éxito

  • El formulario tiene los 6 tipos de input/control vistos en la presentación.
  • Los dos radio buttons son mutuamente excluyentes (porque comparten el mismo name).
  • Al hacer clic en "Enviar inscripción" el navegador intenta enviar el formulario a procesar.php.
  • El documento valida sin errores en W3C.
💡
Pista general: cuando dudes sobre una etiqueta o atributo, la fuente oficial siempre es MDN Web Docs. Un desarrollador profesional la consulta varias veces al día.
